Standard Usage

  • Select environments from the sidebar in the app.
  • Double‑click an environment to rename; duplicates are prevented.
  • Add key/value rows with Add Pair; use Trash to remove.
  • Open YAML imports a file in the exported structure.
  • Export YAML downloads the current configuration as config.yaml.
  • New Config restores the four defaults: Development, QA, Staging, Production.

Keyboard Shortcuts

  • Cmd/Ctrl + N — New Config
  • Cmd/Ctrl + O — Open YAML
  • Cmd/Ctrl + E — Export YAML
  • Cmd/Ctrl + Enter — Add Pair
  • Alt + ↑ / ↓ — Switch environment
  • ? — Open this Knowledge Base

YAML Format

environments:
  Development:
    API_URL: "https://dev.example.com"
    FEATURE_FLAG: "true"
  QA:
    API_URL: "https://qa.example.com"
  Staging:
    API_URL: "https://staging.example.com"
  Production:
    API_URL: "https://api.example.com"

Best Practices

  • Keep environment names consistent across teams and pipelines.
  • Use clear, descriptive keys; avoid secrets in exported files.
  • Store exported YAML under version control with reviews.
  • Validate URLs and flags before deploying changes.
